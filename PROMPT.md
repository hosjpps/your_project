# МЕГА-ПРОМПТ: Лендинг оконной компании «СВОЙ ПРОЕКТ» (г. Выкса)

---

## КОНТЕКСТ И БИЗНЕС-ЗАДАЧА

Создать современный, высококонверсионный одностраничный лендинг для оконной компании «СВОЙ ПРОЕКТ» из города Выкса (Нижегородская область, население ~60 000 человек).

**Компания**: работает с 2008 года, имеет собственное производство ПВХ и алюминиевых конструкций. Хорошо зарекомендовала себя в Выксе и Нижегородской области.

**Текущая проблема**: старый сайт на конструкторе Nethouse (oknavyksa.nethouse.ru) — устаревший дизайн, низкое доверие, не конвертирует посетителей.

**Цель нового лендинга**: максимальная конверсия в заявки/звонки. Целевое действие — оставить заявку на бесплатный замер или позвонить.

---

## ЦЕЛЕВАЯ АУДИТОРИЯ

- **Возраст**: 35-55 лет (основная масса), есть и 25-35
- **Гео**: город Выкса и ближайшие населённые пункты Нижегородской области
- **Уровень владения интернетом**: средний и ниже среднего. Не «диджитал нейтивы». Им нужно всё просто и понятно.
- **Поведение**: предпочитают ЗВОНИТЬ, а не писать. Доверяют «живым» фото и отзывам. Ценят конкретику — цены, сроки, гарантии. Не любят сложные интерфейсы.
- **Устройства**: преимущественно смартфоны (Android), часть — десктоп
- **Как находят**: Яндекс поиск «окна Выкса», «двери Выкса», Яндекс.Карты, 2ГИС, сарафанное радио

---

## СТЕК ТЕХНОЛОГИЙ

- **React** (Vite) + **TypeScript**
- **Tailwind CSS** для стилей
- Компонентная архитектура: каждый блок лендинга = отдельный компонент
- Модальные окна (попапы) для карточек услуг — без сторонних библиотек, на чистом React state + CSS transitions
- Адаптивный дизайн: mobile-first
- Плавный скролл к секциям по якорям
- Анимации: минимальные, ненавязчивые (fade-in при скролле через Intersection Observer)
- Иконки: Lucide React или Heroicons
- Шрифты: Inter (основной) + вариативные веса

```
Успех/зелёный:          #10B981 — для галочек, «бесплатно», гарантий
```

### Принципы дизайна

1. **Много воздуха** — щедрые отступы (py-16, py-20, py-24), контент не слипается
2. **Крупные шрифты** — заголовки от 2xl до 5xl, body минимум 16px (для аудитории 35+)
3. **Контрастные CTA** — янтарные кнопки на синем/белом фоне, крупные (py-4 px-8), скруглённые
4. **Чередование фонов** — белый → светло-серый → тёмный → белый (визуальный ритм)
5. **Карточки с тенью** — мягкие shadow-lg, скруглённые углы (rounded-2xl)
6. **Фото — главный герой** — большие, качественные, в реальных интерьерах
7. **Никаких стоковых улыбающихся людей** — только реальные работы или качественные предметные фото окон/дверей

### Анимация появления при скролле (Intersection Observer)

```css
/* Базовый класс — элемент невидим */
.fade-in-up {
  opacity: 0;
  transform: translateY(30px);
  transition: opacity 0.6s ease-out, transform 0.6s ease-out;
}

/* Класс добавляется когда элемент входит в viewport */
.fade-in-up.visible {
  opacity: 1;
  transform: translateY(0);
}
```

**Настройки Intersection Observer:**
```typescript
const observerOptions = {
  root: null,           // viewport
  rootMargin: '0px',
  threshold: 0.15,      // 15% элемента видно — запускаем анимацию
};
```

### Счётчик статистики (Hero)

Числа «3000+», «15+» анимируются от 0 до конечного значения при появлении в viewport. Длительность: 2 секунды. Easing: ease-out. Реализация — через requestAnimationFrame.

```typescript
function animateCounter(element: HTMLElement, target: number, duration: number = 2000) {
  let start = 0;
  const startTime = performance.now();

  function update(currentTime: number) {
    const elapsed = currentTime - startTime;
    const progress = Math.min(elapsed / duration, 1);
    const eased = 1 - Math.pow(1 - progress, 3); // ease-out cubic
    const current = Math.floor(eased * target);
    element.textContent = current.toLocaleString('ru-RU');
    if (progress < 1) requestAnimationFrame(update);
    else element.textContent = target.toLocaleString('ru-RU') + '+';
  }

  requestAnimationFrame(update);
}
```

## ДЕТАЛЬНОЕ ПОВЕДЕНИЕ ИНТЕРАКТИВНЫХ ЭЛЕМЕНТОВ

### Header — Scroll Spy

Навигация в Header подсвечивает активный пункт в зависимости от текущей видимой секции:

```typescript
// useScrollSpy.ts
const useScrollSpy = (sectionIds: string[], offset: number = 100) => {
  const [activeId, setActiveId] = useState<string>('');

  useEffect(() => {
    const observer = new IntersectionObserver(
      (entries) => {
        entries.forEach((entry) => {
          if (entry.isIntersecting) {
            setActiveId(entry.target.id);
          }
        });
      },
      { rootMargin: `-${offset}px 0px -50% 0px` }
    );

    sectionIds.forEach((id) => {
      const el = document.getElementById(id);
      if (el) observer.observe(el);
    });

    return () => observer.disconnect();
  }, [sectionIds, offset]);

  return activeId;
};
```

Активный пункт меню: текст акцентного цвета + подчёркивание снизу (border-bottom 2px accent).

```typescript
  setTouchStartX(e.touches[0].clientX);
};

const handleTouchEnd = (e: TouchEvent) => {
  if (touchStartX === null) return;
  const diff = touchStartX - e.changedTouches[0].clientX;
  if (Math.abs(diff) > 50) {  // минимальный свайп 50px
    if (diff > 0) next();
    else prev();
  }
};
```

## МАСКА ТЕЛЕФОНА — РЕАЛИЗАЦИЯ

Все поля ввода телефона на сайте должны иметь единую маску:

```typescript
function formatPhone(value: string): string {
  const digits = value.replace(/\D/g, '');
  const match = digits.match(/^(\d{0,1})(\d{0,3})(\d{0,3})(\d{0,2})(\d{0,2})$/);

  if (!match) return value;

  let result = '+7';
  if (match[2]) result += ` (${match[2]}`;
  if (match[2]?.length === 3) result += ')';
  if (match[3]) result += ` ${match[3]}`;
  if (match[4]) result += `-${match[4]}`;
  if (match[5]) result += `-${match[5]}`;

  return result;
}
```

Input type: `tel`. Placeholder: `+7 (___) ___-__-__`. Автозаполнение: `autocomplete="tel"`.

## EDGE-КЕЙСЫ И НЮАНСЫ

### Попапы
- При открытии попапа — `body overflow: hidden` (запрет скролла страницы)
- При закрытии — восстановление скролла
- На мобильном попап занимает почти весь экран (inset-x-2, inset-y-4)
- Фокус-трап: Tab циклится только внутри попапа
- При открытии — фокус на первый интерактивный элемент внутри
- При закрытии — фокус возвращается на кнопку, которая открыла попап

### Фиксированная мобильная панель
- Скрывается при скролле вниз (чтобы не мешала контенту)
- Появляется при скролле вверх (пользователь ищет навигацию)
- Реализация через отслеживание направления скролла (prevScrollY vs currentScrollY)
- z-index: 50 (выше контента, ниже модалок z-index: 60)

### Якорная навигация
- Плавный скролл: `behavior: 'smooth'`
- Учитывать высоту фиксированного header (offset -80px)
- На мобильном: учитывать и нижнюю панель
- После перехода по якорю — обновить URL hash (history.pushState)

### Lazy loading изображений
- Все img ниже первого экрана: `loading="lazy"`
- Hero-фото: `loading="eager"`, `fetchPriority="high"`
- Placeholder до загрузки: серый фон с пульсацией (animate-pulse)


#### `mcp__gsap-master__understand_and_create_animation`
**Кастомные анимации** (context: `"react"`):
```
request: "fade in cards one by one when scrolling into viewport"
request: "smooth counter animation for statistics numbers"
request: "hero text entrance with staggered letters"
request: "modal open/close with scale and fade animation"
complexity: "intermediate"
```

#### Где применять GSAP:
- **Hero-секция**: появление заголовка, подзаголовка и CTA с задержкой
- **Счётчики**: анимация чисел «18 лет опыта», «5000+ объектов»
- **Карточки услуг**: stagger-анимация при скролле
- **Секция преимуществ**: последовательное появление блоков
- **Попапы**: плавное открытие/закрытие с scale-эффектом

#### Ограничения:
- НЕ использовать тяжёлые анимации — ЦА на слабых устройствах
- Анимации должны быть **subtle** — усиливать, а не отвлекать
- Длительность: 0.3-0.8s максимум
- Использовать `will-change` и `transform` для производительности

## ДОПОЛНИТЕЛЬНЫЕ УКАЗАНИЯ

- **НЕ использовать** Lorem ipsum — весь текст должен быть реальным, на русском языке, релевантным бизнесу
- **Placeholder-изображения** — использовать серые блоки с текстом/иконкой и подписью «Фото будет добавлено» или градиентные заглушки
- **Формы** — пока без бэкенда, при отправке показывать success-сообщение «Спасибо! Мы перезвоним вам в течение 15 минут»
- **Попапы услуг** — контент для каждого попапа должен быть содержательным, с реальной информацией о типах продукции, преимуществах, применении
- **Анимации (GSAP)** — subtle, производительные. Fade-in при скролле, stagger для карточек, плавные счётчики. НЕТ: parallax, бесконечные лупы, тяжёлые эффекты
- **Карта** — Яндекс.Карты embed iframe на адрес «Выкса, Красные Зори 31»
- **shadcn/ui** — использовать как основу для ВСЕХ UI-элементов (кнопки, формы, модалки, карточки)
- **21st.dev** — для КАЖДОГО крупного блока получить премиум-компонент, адаптировать под контент
- **GSAP** — анимировать hero, карточки, счётчики, секции при скролле
- **Логотипы** — `mcp__magic__logo_search` для реальных SVG-логотипов REHAU, Veka, KBE
- **Рефайн** — каждый компонент прогнать через `component_refiner`
- **Полировка** — после сборки применить `/polish`, `/adapt`, `/audit`
- **Уникальность** — избегать типичного "AI-дизайна". Каждая секция уникальна и профессиональна
